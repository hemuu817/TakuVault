TRPGセッションツール向け、クラウド素材保管庫「TakuVault」


##サービス概要
TakuVault は、TRPGセッションツールで使用した背景・カットイン・BGM等の素材をクラウド上に保存し、 「どのセッション / どのシーン / どの役割で使った素材か（Where used）」を逆引きできる素材保管庫です。 ※MVPでは Cocofoliaのみ を対象にし、部屋連携は room_urlの参照保存（同期やAPI連携は行わない）とします。


##このサービスへの思い・作りたい理由
趣味のTRPGのKPとして部屋を作る中で、使用した素材を手元で雑に管理してしまい、 「いつ・どのシーンで使った素材か分からない」「良かったBGMや背景が見つからない」という問題が発生することがありました。 既存ツールにも素材保存機能はある一方、シーン単位での分割管理 / リネーム / タグ付け / 逆引き検索がなく、その点を不満に感じています。 そこで、複数セッション・複数ツールに跨って素材を整理できる「クラウドの素材保管庫」を作りたいと考えました。


##ユーザー層について
* TRPGのKP/GMで、セッション準備の素材管理を効率化したいユーザーをターゲットにしています。
* ライトユーザー：素材をクラウドに分かりやすくまとめておきたい
* ヘビーユーザー：複数セッション/複数ツールの素材を一元管理したい


##サービスの利用イメージ
* セッションのURLリンクを貼り付けることで、アプリケーションからセッションにリダイレクトできます。
* 素材は一括でドラッグ&ドロップで投下することができます。
* 投下された素材は拡張子、音声素材の長さ等を判定し、自動的にタグ登録がされます。
* 登録された素材を新たにリネーム、登録されたタグの一括で選択し、追加と変更を行うことができます。
* 元のファイル名またはリネーム後の名前で検索でき、タグの属性、範囲指定によるフィルタリング、各項目に　よるソートをすることができます。
* 保存した素材は、使用したツール・セッションを跨いで ドラッグ&ドロップで移動できます。


##ユーザーの獲得について
* SNS（X など）でTRPG関連のハッシュタグを用いて発信し、特にセッション準備を行うKP/GM層に向けて、以下を主軸に訴求します。
* セッション素材（背景/カットイン/BGM等）を 一元管理できること
* 「どのセッション / どのシーン / どの役割で使ったか」を追跡できること（Where used）
* 次回以降のセッション準備における 検索・再利用の効率化


##サービスの差別化ポイント・推しポイント
TakuVaultの最大の特徴は、素材を単に保存するだけでなく、 Where used（素材がどのセッション/シーン/役割で使われたか）を記録し、素材側から逆引きできる点です。 このアプリケーションでは、「あの時の背景/BGMどこに保存したかな？」の疑問を即座に検索、逆引き機能によって解決できます。 また、将来的には複数のセッションツール（Cocofolia以外）にも対応し、ツールを跨いだ素材管理を可能にすることで継続利用の価値を高めていく想定です。


##機能候補
* ユーザー登録 / ログイン機能（Devise）を搭載し
　ユーザーごとにアカウントと素材を紐づけて管理できるようにします。
* MVPリリース時点では、対応するセッションツールは Cocofolia のみとします。
  (その他のセッションツール対応は本リリース前に、ユドナリウム、TRPGスタジオ等を検討中)
* ルーム連携は同期ではなく、セッションツールの部屋URL（room_url）を保存して参照できる形に留めます。
* MVPでは、素材（Asset）と使用情報（Usage）を中心に
　Where used の価値が最低限成立する実装をします。
* 素材の一括アップロード機能（ドラッグ&ドロップ対応）
* 素材の自動タグ付け機能（拡張子(jpg, png, mp3, wav, pdf等を判別)、音声素材の長さ等に基づく）
* 素材のリネーム、タグ編集機能（複数選択対応）
* 素材の検索・フィルタリング・ソート機能
* 素材の移動（セッション/シーン/役割間のドラッグ&ドロップ対応）
* また、MVPでは見送る想定ですが、投下した動画素材を APNG+MP3に変換し、Cocofolia等でカットインとし　てすぐ利用できる機能も検討しています。


##使用する技術スタック
* 使用するフレームワーク: Ruby 3.x, Rails 7.x（フルスタック）
* データベース: PostgreSQL
* デプロイ先: Render（Web：Dockerデプロイ）
* ストレージ / ライブラリ: ActiveStorage, 本番ストレージ：Cloudflare R2（S3互換）を利用予定
